<objective>
Fix src/server/lib/openai.ts to use GPT-4o Structured Outputs (json_schema) instead of json_object, add Zod validation as a safety net, pin the model version, and enforce log discipline.

Purpose: The current wrapper uses json_object mode, which returns valid JSON but never enforces field types or enum values. A single malformed GPT-4o response propagates silently into selectBlocks. This fix adds two enforcement layers: the API-level schema (refuses non-conforming output) and a Zod parse (TypeScript narrowing + clear error path).
Output: Updated openai.ts with HAND_ATTRIBUTES_SCHEMA constant, HandAttributesSchema (Zod), and a hardened analyzeHand() function.
</objective>
