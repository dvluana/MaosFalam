**State management:**

- `useAuth()` for user/hydrated
- `useCredits()` for balance/reading_count/loading
- Local state: `name`, `email`, `gender` ("female"|"male"), `dominantHand` ("right"|"left"), `emailOptIn`, `isSelf` (boolean, default true), `showCreditGate` (boolean), `submitting`, `confirming`, `emailError`

**Flow A — Visitor (user === null):**
Shows full form:

1. Name field (Input component, label: "Seu nome", placeholder: "O primeiro que vier")
2. Email field (Input component, label: "Melhor email", placeholder: "voce@exemplo.com")
3. Gender toggle: Ela/Ele (two buttons, same style as current)
4. Dominant hand toggle: Destra/Canhota (two buttons, same style as gender toggle)
   - Label (Cormorant italic 14px, bone-dim): "Qual mao voce usa mais?"
   - "Destra" = right, "Canhota" = left, default: "Destra" (right)
5. LGPD opt-in checkbox (same as current)
6. Submit: validates name >= 2 chars, email contains @, then:
   a. Generates session_id (crypto.randomUUID)
   b. Saves to sessionStorage: maosfalam_name, maosfalam_email, maosfalam_name_fresh=1, maosfalam_target_gender
   c. Calls `registerLead()` (fire-and-forget, failure does NOT block — per CTX-09)
   d. Builds ReadingContext: { target_name, target_gender: gender, dominant_hand: dominantHand, is_self: true, session_id, credit_used: false }
   e. Calls `saveReadingContext(ctx)`
   f. Navigates to /ler/toque

**Flow B — Logged-in (user !== null):**
Shows different UI:

1. Heading (Cormorant italic 28px): "Pra quem e essa leitura?"
2. Two toggle buttons (same style as gender):
   - "Pra mim" — sets isSelf=true, pre-fills name from user.name, hides name/email/gender fields
   - "Pra outra pessoa" — sets isSelf=false, shows name + gender fields (email NOT needed — already logged in)
3. When isSelf=true: only show dominant hand toggle + submit button
4. When isSelf=false: show name field + gender toggle + dominant hand toggle + submit button
5. No email field, no LGPD opt-in (already has account)
6. Submit logic:
   a. If `reading_count === 0`: first reading is free. Build ReadingContext with credit_used=false. Save. Navigate to /ler/toque. (per CTX-06)
   b. If `reading_count > 0 && balance > 0`: show CreditGate modal (set showCreditGate=true). (per CTX-07, CTX-05)
   c. If `reading_count > 0 && balance === 0`: redirect to /creditos. (per CTX-07)

**CreditGate flow (when showCreditGate=true):**

- Renders `<CreditGate>` overlay with balance, targetName, confirming state
- onConfirm: calls `requestNewReading({ target_name, target_gender, is_self })` then:
  - On success: build ReadingContext with credit_used=true, save, navigate to /ler/toque
  - On 402 error: redirect to /creditos
- onCancel: close modal (set showCreditGate=false)

**Visual layout:**

- Keep existing atmosphere div (radial gradient)
- Keep existing eyebrow ("Antes de eu ler" badge)
- For logged-in flow, change the heading from "Me diz duas coisas antes." to "Pra quem e essa leitura?" (Cormorant italic)
- Subheading for logged-in: remove (not needed)
- For visitor, keep current headings: "Me diz duas coisas antes." / "Como eu te chamo, e onde eu te encontro depois."
- All toggle buttons (gender, dominant hand, pra mim/outra) use identical styling: the branded button style with gradient background when selected, transparent when not, `border-radius: 0 6px 0 6px`

**While credits are loading (logged-in only):**

- Disable submit button, show normal form (no skeleton needed — the form is usable, just can't submit yet)

**Important constraints:**

- Do NOT remove lead registration for visitors (CTX-09)
- Do NOT call requestNewReading for visitors or first-time users (CTX-06, CTX-08)
- Dominant hand default: "right" (Destra)
- No `any` types
- Use `import type { ReadingContext } from "@/types/reading-context"`
