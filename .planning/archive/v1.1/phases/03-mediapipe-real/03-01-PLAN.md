   a) loadHandLandmarker(): Promise<HandLandmarker>
   - Import { FilesetResolver, HandLandmarker } from "@mediapipe/tasks-vision"
   - Create HandLandmarker with:
     - baseOptions.modelAssetPath: use CDN path "https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/latest/hand_landmarker.task"
     - baseOptions.delegate: "GPU"
     - runningMode: "VIDEO"
     - numHands: 1
   - Return the landmarker instance

   b) ValidLandmarkResult type:

   ```typescript
   export interface ValidLandmarkResult {
     isHandPresent: boolean;
     isOpen: boolean;
     isCentered: boolean;
     detectedHandedness: "Left" | "Right";
   }
   ```

   c) validateLandmarks(landmarks: NormalizedLandmark[], frameWidth: number, frameHeight: number): ValidLandmarkResult
   - isHandPresent: landmarks.length >= 21
   - isOpen: distance between THUMB_TIP (4) and PINKY_TIP (20) > 0.35 \* distance(WRIST(0), MIDDLE_MCP(9))
     (palm height as reference, fingers must be spread wide enough)
   - isCentered: WRIST (0) and MIDDLE_MCP (9) x coords between 0.15 and 0.85, y coords between 0.10 and 0.90
     (normalized coordinates, so 0-1 range)
   - detectedHandedness: from the HandLandmarker result category (not from landmarks)
     NOTE: Will be passed separately from the handedness result, not computed from landmarks

   d) detectHandedness(handedness: Category[]): "Left" | "Right"
   - MediaPipe returns handedness as a Category with categoryName "Left" or "Right"
   - Return the categoryName of the first result
   - IMPORTANT: MediaPipe reports handedness from the camera's perspective. For a MIRRORED front camera, MediaPipe's "Right" actually means the user's RIGHT hand. For a non-mirrored back camera, MediaPipe's "Left" means the user's LEFT hand. The caller (useCameraPipeline) handles this mapping.

   e) captureFrame(video: HTMLVideoElement, canvas: HTMLCanvasElement, mirrored: boolean): string
   - Set canvas dimensions to video.videoWidth x video.videoHeight
   - If mirrored: ctx.translate(canvas.width, 0); ctx.scale(-1, 1) — so the captured photo is NOT mirrored (natural orientation for GPT-4o analysis)
   - Draw video frame to canvas
   - Return canvas.toDataURL("image/jpeg", 0.92) and strip the "data:image/jpeg;base64," prefix to return raw base64

   Use "use client" directive at top since this imports browser-only MediaPipe.

<task type="auto">
  <name>Task 2: Rewrite useCameraPipeline with real MediaPipe detection loop</name>
  <files>src/hooks/useCameraPipeline.ts</files>
  <action>
Completely rewrite useCameraPipeline. The new hook signature adds videoRef and canvasRef:

```typescript
interface Params {
  state: CamState;
  forced: boolean;
  setState: (s: CamState) => void;
  onCaptured: (photoBase64: string) => void;
  videoRef: React.RefObject<HTMLVideoElement | null>;
  canvasRef: React.RefObject<HTMLCanvasElement | null>;
}
```

Implementation:

1. **State: loading_mediapipe**
   When state === "loading_mediapipe" and NOT forced:
   - Load ReadingContext from sessionStorage via loadReadingContext() to get dominant_hand
   - Call getUserMedia({ video: { facingMode: "environment", width: { ideal: 1280 }, height: { ideal: 960 } } })
     - On success: attach stream to videoRef.current.srcObject, set mirrored flag based on track settings (facingMode === "user" means mirrored)
     - On NotAllowedError: setState("camera_permission_denied")
     - On other errors: setState("camera_error_generic")
   - Load HandLandmarker via loadHandLandmarker()
   - When both video playing and landmarker ready: setState("camera_active_no_hand")
   - Store landmarker instance and mirrored flag in refs (useRef)

2. **Detection loop (requestAnimationFrame)**
   When state is camera_active_no_hand, camera_hand_detected, camera_adjusting, camera_wrong_hand, or camera_stable:
   - Call landmarker.detectForVideo(video, performance.now())
   - If no hand detected: setState("camera_active_no_hand"), reset stability timer
   - If hand detected:
     - Get handedness via detectHandedness(result.handednesses[0])
     - Map handedness to user perspective:
       - Front camera (mirrored): MediaPipe "Left" = user's Left, "Right" = user's Right (mirroring cancels out)
       - Back camera (not mirrored): MediaPipe "Left" = user's Right, "Right" = user's Left (mirror flip)
     - Compare with dominant_hand from ReadingContext:
       - User's dominant_hand is "left" or "right"
       - We want to see the OPPOSITE hand (you read the non-dominant palm in quiromancia? Actually no — the docs say dominant_hand is collected. Check: architecture.md says "Handedness (destra/canhota) collected before camera". The camera should photograph the hand that matches dominant_hand. So if dominant_hand is "right", we expect to see the right hand.)
       - If detected hand doesn't match expected: setState("camera_wrong_hand"), continue loop
     - Call validateLandmarks on result.landmarks[0]
     - If NOT isOpen or NOT isCentered: setState("camera_adjusting"), reset stability timer
       - Refine feedback: if not isOpen, set a detail ref "Abra mais os dedos." If not isCentered, "Centralize a mao."
     - If isOpen AND isCentered AND hand matches:
       - setState("camera_hand_detected") if not already hand_detected/stable
       - Increment stability counter (track consecutive valid frames)
       - If stable for 1.5s (~45 frames at 30fps, but use timestamp-based check):
         setState("camera_stable")
       - After entering camera_stable, wait 0.5s more, then:
         setState("camera_capturing")
         vibrate(120)
         Call captureFrame(video, canvas, mirrored)
         Call onCaptured with the base64 result

3. **Cleanup**
   On unmount or when state leaves detection range:
   - Cancel requestAnimationFrame
   - Stop all video tracks (stream.getTracks().forEach(t => t.stop()))
   - Close HandLandmarker

4. **Forced mode (preview/dev)**
   If forced is true, skip all MediaPipe logic (same as before — dev preview mode using StateSwitcher).

Key implementation details:

- Use useRef for: landmarker instance, stream, animationFrameId, stabilityStartTime, mirrored flag, dominant_hand
- Use useEffect with cleanup for the loading phase
- Use a separate useEffect for the detection loop (depends on state being in active range)
- The stability check uses timestamps: record Date.now() when first valid frame detected, check if (now - stabilityStart) >= 1500
- After capture, stop the detection loop but don't stop the stream yet (let the page handle navigation)
- DO NOT use console.log/warn/error (eslint no-console rule). If you need to handle errors silently, just setState to error states.
