  <action>
    Audit finding: all `logger.error({ error, ... })` calls pass the raw `error` object.
    Prisma errors can embed query parameters (including user-provided values) in their
    message or cause string. Raw Error objects also expose full stack traces that may
    contain interpolated data.

    Fix: replace `error` with `err: error instanceof Error ? error.message : String(error)`
    in EVERY logger.error call that includes the error object.

    Pattern to find: `logger.error({ error,` or `logger.error({ err,`

    Apply this change to every matching call in these files:
    - src/app/api/lead/register/route.ts — line ~57
    - src/app/api/reading/capture/route.ts — line ~121
    - src/app/api/reading/[id]/route.ts — line ~43
    - src/app/api/reading/[id]/upgrade/route.ts — line ~73
    - src/app/api/credits/purchase/route.ts — line ~101
    - src/app/api/user/claim-readings/route.ts — line ~64
    - src/app/api/user/credits/route.ts — line ~52
    - src/app/api/user/profile/route.ts — lines ~27 and ~66
    - src/app/api/user/readings/route.ts — line ~72
    - src/app/api/webhook/abacatepay/route.ts — line ~136
    - src/server/lib/abacatepay.ts — line ~19
    - src/server/lib/resend.ts — line ~32 (`{ err }`)

    Example transformation:
    BEFORE: `logger.error({ error, route: "/api/lead/register" }, "Erro na rota");`
    AFTER:  `logger.error({ err: error instanceof Error ? error.message : String(error), route: "/api/lead/register" }, "Erro na rota");`

    For src/server/lib/resend.ts line 32 (`logger.error({ err }, "Resend fetch failed")`):
    AFTER: `logger.error({ err: err instanceof Error ? err.message : String(err) }, "Resend fetch failed");`

    Do NOT change logger.warn or logger.info calls — those already log only IDs and
    safe metadata (per audit: readingId, clerkUserId, packId, element, confidence,
    billingId, leadId, packType, variation, axis, to_hash, subject — all safe).

    After editing, run type-check to confirm no TypeScript errors:
    `npm run type-check`
