**Tipos exportados:**

```typescript
export type CheckStatus = "pending" | "running" | "pass" | "fail" | "skip";

export interface ValidationCheck {
  id: "format" | "size" | "hand_detected" | "handedness" | "palm_open";
  label: string; // texto pro usuario (ex: "Formato aceito")
  status: CheckStatus;
  detail?: string; // detalhe opcional (ex: "JPEG detectado")
}

export interface ValidationResult {
  phase: "idle" | "validating" | "done";
  checks: ValidationCheck[];
  previewUrl: string | null;
  file: File | null;
  // Resultado final
  handOk: boolean; // mao detectada e handedness correto
  qualityOk: boolean; // palma aberta e tamanho ok
  canProceed: boolean; // handOk=true (mesmo se qualityOk=false)
  error: string | null; // erro fatal que impede prosseguir
}
```

**Assinatura do hook:**

```typescript
export function useUploadValidation(dominantHand: "right" | "left"): {
  result: ValidationResult;
  validate: (file: File) => Promise<void>;
  reset: () => void;
};
```

**Logica de validacao (validate function) — 5 checks em sequencia:**

CHECK 1 — formato:

- label: "Formato aceito"
- ACEITAR: image/jpeg, image/png, image/webp, image/heic, image/heif
- REJEITAR (erro fatal, para aqui): image/gif, image/svg+xml, image/bmp, application/pdf, e qualquer outro
- Se rejeitado: check.status = "fail", result.error = "Preciso de uma foto. JPG, PNG, WebP ou HEIC."
- Se aceito: check.status = "pass", check.detail = tipo detectado em uppercase (ex: "JPEG")

CHECK 2 — tamanho:

- label: "Tamanho ok"
- Max: 15MB (15 _ 1024 _ 1024 bytes)
- Se maior: check.status = "fail", result.error = "A foto e muito grande. Use uma menor que 15MB."
- Se ok: check.status = "pass", check.detail = tamanho formatado (ex: "2.4 MB")

CHECK 3 — mao detectada (MediaPipe):

- label: "Mao na foto"
- Tentar carregar MediaPipe. Se falhar (rede, timeout), check.status = "skip" e continuar.
- Se MediaPipe carregou: criar HTMLImageElement com src = URL.createObjectURL(file), aguardar onload
- Criar HandLandmarker com runningMode "IMAGE" (ver NOTA CRITICA acima — usar FilesetResolver + HandLandmarker.createFromOptions direto com runningMode: "IMAGE")
- Chamar landmarker.detect(imgElement)
- Se landmarks.length === 0: status = "fail", handOk = false, error = "Nao encontrei uma mao na foto. Tente de novo."
- Se detectou: status = "pass"

CHECK 4 — handedness (mao correta):

- label: "Mao correta"
- So executa se check 3 passou
- Para uploads (nao espelhado): "Right" do MediaPipe = mao direita real, "Left" = esquerda real
- Comparar com dominantHand ("right"/"left")
- Se errada: status = "fail", handOk = false, error = "Essa e a mao {errada}. Me manda a {dominantHand === 'right' ? 'direita' : 'esquerda'}."
- Se correta: status = "pass", handOk = true

CHECK 5 — palma aberta:

- label: "Palma aberta"
- So executa se check 3 passou (mesmo que check 4 falhou)
- Usar validateLandmarks(landmarks[0], imgElement.naturalWidth, imgElement.naturalHeight)
- Se !isOpen: status = "fail", qualityOk = false (NAO e erro fatal)
- Se isOpen: status = "pass", qualityOk = true

**canProceed**: true se handOk = true (qualityOk pode ser false — fluxo "Usar mesmo assim")
