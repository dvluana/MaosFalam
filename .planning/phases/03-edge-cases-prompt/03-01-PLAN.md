  <action>
Criar o modulo com a funcao `normalizeImage(file: File): Promise<File>`.

A funcao executa 3 etapas em sequencia:

**Etapa 1 â€” HEIC para JPEG (EDGE-01):**

- Verificar se `file.type === "image/heic"` ou `file.type === "image/heif"` ou `file.name.toLowerCase().endsWith(".heic")` ou `.heif`
- Se sim: `const heic2any = (await import("heic2any")).default`
- Chamar `heic2any({ blob: file, toType: "image/jpeg", quality: 0.9 })` â€” retorna `Blob | Blob[]`
- Se retornou array, pegar `[0]`
- Criar `new File([blob], file.name.replace(/\.(heic|heif)$/i, ".jpg"), { type: "image/jpeg" })`
- Continuar com esse File como `file` pra proxima etapa

**Etapa 2 â€” EXIF rotation correction (EDGE-02):**

- Usar `createImageBitmap(file)` â€” browsers modernos aplicam EXIF automaticamente
- Criar um `OffscreenCanvas` ou um `HTMLCanvasElement` (no browser) com as dimensoes corretas do bitmap
- `ctx.drawImage(bitmap, 0, 0)` â€” o bitmap ja esta rotacionado corretamente pelo browser
- Isso garante que o canvas tem a orientacao visual correta mesmo que o EXIF diga "rotacionar"
- Nota: em SSR/server, `createImageBitmap` nao existe â€” proteger com `typeof createImageBitmap !== "undefined"`
- Se nao disponivel (SSR ou browser antigo), retornar o file sem modificacao desta etapa

**Etapa 3 â€” Compressao e resize (EDGE-03):**

- Usar o bitmap (ou carregar de novo se a etapa 2 foi pulada) pra um canvas
- Se `bitmap.width > 1280`: calcular ratio `1280 / bitmap.width`, criar canvas com `width=1280, height=bitmap.height * ratio`
- Se `bitmap.width <= 1280`: usar dimensoes originais
- `ctx.drawImage(bitmap, 0, 0, canvas.width, canvas.height)`
- `canvas.toDataURL("image/jpeg", 0.85)` â†’ converter dataURL de volta pra Blob â†’ `new File([blob], outputName, { type: "image/jpeg" })`
- Converter dataURL pra Blob: `await (await fetch(dataUrl)).blob()`

**Assinatura final:**

```typescript
// "use client" no topo â€” usa canvas/createImageBitmap
export async function normalizeImage(file: File): Promise<File>;
```
