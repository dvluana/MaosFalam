  <name>Task 2: Fix camera handedness for back camera (BUG-02)</name>
  <files>src/hooks/useCameraPipeline.ts</files>
  <action>
    The bug: MediaPipe Hand Landmarker assumes mirrored (selfie/front camera) input. When using the back camera, its handedness labels are inverted. Currently the detection loop (line ~246) maps MediaPipe labels directly without considering camera mode:

    ```typescript
    const userHandedness: "left" | "right" = rawHandedness === "Left" ? "left" : "right";
    ```

    This is correct for front camera (mirrored=true) but WRONG for back camera (mirrored=false). For back camera, MediaPipe "Right" = user's LEFT hand.

    Fix: Use `mirroredRef.current` to determine mapping:

    ```typescript
    // Front camera (mirrored): MediaPipe label matches user's hand directly
    // Back camera (not mirrored): MediaPipe label is inverted
    const userHandedness: "left" | "right" = mirroredRef.current
      ? (rawHandedness === "Left" ? "left" : "right")   // front: direct
      : (rawHandedness === "Left" ? "right" : "left");   // back: invert
    ```

    This does NOT affect:
    - Upload validation (useUploadValidation.ts already inverts at line 288, correct for static photos)
    - Front camera detection (unchanged behavior)
    - captureFrame mirroring (already gated on mirroredRef.current)

    Also update the comment above the handedness check (around line 241-244) to accurately describe the mapping logic for both camera modes. Remove the misleading comment that says "MediaPipe's label matches the user's actual hand regardless of facing mode" since that's the bug.

  </action>
