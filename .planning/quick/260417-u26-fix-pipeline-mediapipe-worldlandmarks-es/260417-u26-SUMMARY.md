## What Changed

### Task 1 — mediapipe.ts core

- Added `dist3D(a, b)` function: euclidean distance in 3D space using `{x,y,z}` (meters)
- Rewrote `computeElementHint` signature: `(worldLandmarks: Array<{x,y,z}>): HandElement | undefined`
  - Removed `frameWidth`/`frameHeight` params entirely (no aspect ratio correction needed)
  - Uses `dist3D()` for all distance calculations
  - Guards: `palmWidth < 0.001` (meter scale, not normalized scale)
  - Threshold: `fingerRatio >= 0.75` (was 0.95 — calibrated for meter-scale values)
- Added `angleDeg` to `ValidLandmarkResult` interface
- Updated `validateLandmarks` to compute and return `angleDeg` from WRIST→MIDDLE_MCP vector
  - Uses `Math.atan2(|dx|, |dy|)` converted to degrees
  - 0 = hand pointing straight up/down, 90 = horizontal
  - Returns `angleDeg: 0` when no hand present
- Exported `HandElement` type (was internal — needed by useCameraPipeline)
- Kept `dist()` (2D, still used for isOpen/isCentered screen-space checks)
- Tests: 13 tests all pass (8 element classification + 4 angle + 1 no-hand)

### Task 2 — useCameraPipeline.ts

New constants:

- `JITTER_THRESHOLD = 0.025` (2.5% max frame-to-frame movement)
- `STABLE_FRAMES_REQUIRED = 5` (minimum frames in jitter buffer)
- `ELEMENT_SAMPLES_REQUIRED = 8` (samples to collect for mode selection)
- `MAX_ANGLE_DEG = 45` (above this triggers camera_adjusting)

New refs:

- `landmarkHistoryRef` — tracks WRIST+MIDDLE_MCP normalized positions across frames
- `elementSamplesRef` — accumulates HandElement samples for mode selection

New helpers:

- `isHandStill()` — checks all consecutive pairs in history for movement < JITTER_THRESHOLD
- `elementMode(samples)` — returns most frequent element (requires >= 3 samples)
- `resetBuffers()` — resets stabilityStart + landmarkHistory + elementSamples

Detection loop changes:

- Any invalid frame (no hand, wrong hand, not open, not centered, angle > 45) calls `resetBuffers()`
- Valid frames: push to `landmarkHistoryRef`, sample element from `worldLandmarks?.[0]` via `computeElementHint`
- Jitter check (`isHandStill()`) must pass before stability timer advances
- On capture: uses `elementMode(elementSamplesRef.current)` instead of single-frame hint
- Removed `lastLandmarksRef` (replaced by `elementSamplesRef`)

camera.ts: `camera_adjusting` feedback changed to "Ajuste a posicao da mao." (covers both open-fingers and angle issues).

### Task 3 — useUploadValidation.ts

- Added static import of `computeElementHint` and `setElementHint`
- After MediaPipe detect: extracts `worldLm = detection.worldLandmarks?.[0] ?? []`
- After palm_open check passes: computes element hint from worldLm (if >= 21 landmarks), stores via `setElementHint`
- Angle validation: `angleDeg > ANGLE_BLOCK_DEG (60)` blocks with "A mao ta muito inclinada" error
- 45-60 degrees: proceeds (more tolerant than camera which blocks at 45)

openai.ts: Verified correct, no changes needed. `elementHintText` is constructed with ternary (null when no hint), conditionally spread into messages array.
